# Phase 1.4f — OpenRouter Cost Accuracy — Design

Task: [T-20260511-003](../../tasks/active/T-20260511-003.md)
Date: 2026-05-11
Status: Draft (awaiting user review)

## 1. Goal

When the Claude Agent SDK runs against OpenRouter's Anthropic-compat
endpoint, `total_cost_usd` in the SDK's `result` message arrives as `0`,
so `TokenUsage.costUsd` ends up `0` and the Phase 1.4e per-user daily
quota is permanently underfilled. This phase fetches the authoritative
cost from OpenRouter's Generation API per LLM call, sums them, and emits
the corrected values in `agent.done` so the existing host persistence
path writes real cost data into `TokenUsage`.

Scope is **claude-code runtime only** for this slice. OpenHands has its
own LiteLLM pipeline; a parallel fix there is deferred.

## 2. Decisions (recap from brainstorming)

| # | Decision | Rationale |
|---|---|---|
| 1 | **Per-generation-ID lookup** via `GET https://openrouter.ai/api/v1/generation?id=<id>`. Activity-window aggregation is impossible because all users share one OpenRouter API key — concurrent runs would steal each other's cost data. | Each `assistant` SDK message carries `message.id` which equals OpenRouter's `generation_id` (`gen-…` prefix). Per-ID lookup is atomic per LLM call and cannot mis-attribute. |
| 2 | **Fix lives in the agent-runner**, NOT in the host. The runner already has env, network, and SDK access; correcting `agent.done` before it leaves the runner means no protocol change, no host code change, no `TokenUsage` schema change. | Closest to the source of truth. Symmetric with other runtimes which already report their own cost. |
| 3 | **Synchronous fetch before `agent.done` emit**: the runner waits for all OpenRouter lookups to complete before forwarding the terminal event. | Phase 1.4e's pre-flight quota check runs at the next turn dispatch; it needs cost to be in `TokenUsage` by then. Async would race. Per-turn latency cost: ~50–200 ms per LLM call × N calls (OpenRouter API is fast). |
| 4 | **Fallback chain on lookup failure**: if any `generation_id` lookup fails (404 / 5xx / network / timeout), fall back to the existing `estimateCostUsd` model-pricing table for that one call. Other calls in the same turn still use real OpenRouter data when their lookups succeed. | No new persistence behaviour; reuses the existing fallback path that already handles "tokens but no cost" today. |
| 5 | **Detection via env at runtime**: the runner reads `process.env.ANTHROPIC_BASE_URL` at turn-start. If it matches `/openrouter\.ai/i` it activates the lookup path. Otherwise the existing behaviour is unchanged. | Single feature flag, no DB column needed. Same env mechanism that already routes the SDK to OpenRouter is the trigger to look up its cost. |
| 6 | **OpenHands runtime is out of scope** for 1.4f. | OpenHands cost reporting goes through LiteLLM which has its own price book and reports `costUsd` directly. The 0-cost symptom may or may not appear there; if it does, it's a parallel fix in `openhands-runner.ts`, not this slice. |

## 3. Architecture

```
Agent-runner (sdk-runner.ts)
  ┊  At turn start: openrouterMode = /openrouter\.ai/i.test(ANTHROPIC_BASE_URL)
  ┊  responseIds: string[] = []
  ▼
  for await (msg of SDK iterator):
    mapResult = mapSdkMessage(msg, ctx)
    if mapResult.captured?.messageId: responseIds.push(it)   ← extend mapper
    forward mapResult.events as usual (assistant tool_use, chunks, etc.)
  ▼
  When SDK's result message arrives:
    mapper returns agent.done with SDK-reported tokens/cost
    if openrouterMode && responseIds.length > 0:
      correctedUsage = await fetchOpenRouterCosts(responseIds, env.OPENROUTER_API_KEY)
      patch agent.done with corrected costUsd, tokensIn, tokensOut
    emit agent.done

Host (executor-client.ts)
  ┊  No change. persistTurnSubtype writes whatever agent.done carried.
  ┊  estimateCostUsd fallback still fires if costUsd === 0 AND tokens > 0
  ┊  (e.g. happens when EVERY OpenRouter lookup failed and we left SDK 0s in).
```

## 4. Schema

**No migration required.** `TokenUsage.costUsd` and the token columns are
already there; we're just filling them with accurate numbers.

Optional debugging aid: a denormalised `TokenUsage.costSource` column
(`"sdk" | "openrouter_api" | "estimate"`) would help post-incident
analysis. **Out of scope for this slice** — defer to a later phase if
the need arises. The existing `rawUsage` JSON field can stash a
provenance flag for now.

## 5. Protocol

**No `@wbd/protocol` bump.** The fix happens before `agent.done` leaves
the runner; the protocol shape is unchanged. The runner just emits
different (correct) numbers.

## 6. Agent-runner changes

### 6.1 `sdk-event-mapper.ts` — capture `assistant` message IDs

Extend `MapResult` to carry an optional `capturedMessageId`:

```ts
export interface MapResult {
  events: BrokerToHost[];
  captured?: { providerSessionId?: string; modelId?: string };
  capturedMessageId?: string;     // NEW: response.id from assistant messages
}
```

In the `type === "assistant"` branch (around line 84), capture the
`message.id` even when there are no `tool_use` content blocks (because
text-only responses also have a cost):

```ts
if (type === "assistant") {
  const message = (msg as { message?: unknown }).message;
  const capturedMessageId = isRecord(message) && typeof (message as { id?: unknown }).id === "string"
    ? (message as { id: string }).id
    : undefined;
  const content = isRecord(message) ? (message as { content?: unknown }).content : undefined;
  if (!Array.isArray(content)) {
    return { events: [], ...(capturedMessageId ? { capturedMessageId } : {}) };
  }
  const events: BrokerToHost[] = [];
  for (const block of content) {
    if (isRecord(block) && (block as { type?: unknown }).type === "tool_use") {
      // …existing tool_use mapping unchanged
    }
  }
  return { events, ...(capturedMessageId ? { capturedMessageId } : {}) };
}
```

Other branches unchanged. Pure-function semantics preserved.

### 6.2 New module `openrouter-cost.ts`

```ts
// container/sandbox/agent-runner/src/openrouter-cost.ts
export interface OpenRouterGenerationCost {
  generationId: string;
  totalCost: number;            // USD; 0 if unavailable
  promptTokens: number;
  completionTokens: number;
  ok: boolean;                  // false → fallback path triggered
}

export interface FetchCostsOptions {
  openrouterApiKey: string;
  fetch?: typeof globalThis.fetch;
  timeoutMs?: number;           // per-call; default 5000
  baseUrl?: string;             // default "https://openrouter.ai/api/v1"
  log?: (msg: string, err?: unknown) => void;
}

export async function fetchOpenRouterGenerationCost(
  generationId: string,
  opts: FetchCostsOptions,
): Promise<OpenRouterGenerationCost>;

export async function fetchOpenRouterCosts(
  generationIds: string[],
  opts: FetchCostsOptions,
): Promise<{
  totalCost: number;
  promptTokens: number;
  completionTokens: number;
  attempted: number;
  succeeded: number;
}>;
```

`fetchOpenRouterCosts` calls the per-ID variant in parallel (`Promise.all`).
Each per-ID failure is silently absorbed (logged, marked `ok: false`,
returns zeros) so the parent aggregate still reflects the calls that
did succeed.

OpenRouter response shape (per their docs):

```json
{
  "data": {
    "id": "gen-...",
    "model": "anthropic/claude-3-5-sonnet",
    "total_cost": 0.0123,
    "native_tokens_prompt": 1234,
    "native_tokens_completion": 567,
    ...
  }
}
```

We use `data.total_cost`, `data.native_tokens_prompt`, and
`data.native_tokens_completion` (more accurate than `tokens_prompt` /
`tokens_completion` which can be normalised across providers).

### 6.3 `sdk-runner.ts` — wire the correction

State held during the turn:

```ts
const openrouterMode = /openrouter\.ai/i.test(process.env.ANTHROPIC_BASE_URL ?? "");
const responseIds: string[] = [];
```

Inside the `for await (const msg of iterator)` loop, after `mapSdkMessage`:

```ts
if (out.capturedMessageId) responseIds.push(out.capturedMessageId);
```

For each event in `out.events`, if it's `agent.done` AND
`openrouterMode === true` AND `responseIds.length > 0`:

```ts
const openrouterApiKey = process.env.OPENROUTER_API_KEY ?? "";
if (openrouterApiKey) {
  const corrected = await fetchOpenRouterCosts(responseIds, {
    openrouterApiKey,
  });
  if (corrected.succeeded > 0) {
    event = {
      ...event,
      tokensIn: corrected.promptTokens,
      tokensOut: corrected.completionTokens,
      costUsd: corrected.totalCost,
    };
    // Also stash provenance into the usage payload's rawUsage if present.
  }
}
emit(event);
```

If `corrected.succeeded === 0` (all lookups failed), we leave the SDK's
values in place. The host's existing `estimateCostUsd` fallback then
fills in a price-table estimate when it sees `costUsd === 0 && tokens > 0`.

## 7. Host changes

**None.** The existing `persistTurnSubtype` in `executor-client.ts`:
1. Reads `event.costUsd` from `agent.done` → now corrected.
2. Falls back to `estimateCostUsd(...)` when `costUsd === 0 && tokens > 0`.
3. Writes `TokenUsage` row.

Phase 1.4e's `getUserDailySpend` aggregates whatever's in `TokenUsage`.

## 8. Edge cases

| Case | Behaviour |
|---|---|
| Not using OpenRouter (`ANTHROPIC_BASE_URL` unset or anthropic.com) | `openrouterMode = false`. SDK values flow through unchanged. Zero overhead. |
| `OPENROUTER_API_KEY` env unset but base URL matches | Logged warning; no lookups attempted; SDK values flow through (likely 0); host's `estimateCostUsd` fills in. |
| All `assistant` messages had no `id` (unusual SDK version) | `responseIds.length === 0`. Skip the lookup. SDK values flow through. |
| Some lookups succeed, some fail | Aggregate of successful ones is the corrected `costUsd`. Tokens are summed from successful ones too. Failed ones contribute zero. Host's estimate fallback can't help here (tokens are already > 0 from successful ones) — the gap is silent under-reporting, acceptable. |
| All lookups fail (e.g. OpenRouter outage) | SDK values flow through (`costUsd === 0`). Host's `estimateCostUsd` then computes from `tokensIn` / `tokensOut` (also possibly 0 if OR strips those too). Quota under-reports for the turn; logged. |
| OpenRouter slow / timeout | Per-call timeout 5 s; total turn-end latency at most `5s × N parallel calls = 5s` (Promise.all is parallel). Acceptable. |
| Same `generation_id` queried twice (resume?) | Idempotent — OpenRouter returns the same record. No double-counting because IDs are deduplicated as map keys client-side if needed (currently not — duplicate IDs from same SDK iterator are rare; ignored). |
| Concurrent runs same OpenRouter key (multi-user) | Each turn only queries its OWN response IDs, captured per-iterator. No cross-contamination. |

## 9. Test plan

- **`sdk-event-mapper.ts` unit tests**: `assistant` message with `id` → `capturedMessageId` set; `assistant` without `id` → undefined; existing tool_use / text behaviour unchanged.
- **`openrouter-cost.ts` unit tests** (mocked `fetch`):
  - happy path: 200 with JSON body → returns parsed values, `ok: true`
  - 404 → `ok: false`, zeros
  - 5xx → `ok: false`, zeros
  - timeout → `ok: false`, zeros
  - empty `generationIds` array → returns zero-shaped aggregate
  - 2 ok + 1 fail → aggregate counts 2 successes, total cost = sum of 2
- **`sdk-runner.ts` integration test** (mocked SDK iterator + mocked `fetchOpenRouterCosts`): given `ANTHROPIC_BASE_URL=https://openrouter.ai/api`, two `assistant` events with IDs, a `result` with `total_cost_usd: 0` → `agent.done` is patched with the mocked OpenRouter aggregate.
- **`sdk-runner.ts` integration test**: given `ANTHROPIC_BASE_URL=https://api.anthropic.com`, openrouter lookup is NOT invoked, SDK values flow through.
- No host-side tests needed — host is unchanged.

## 10. Out of scope (for 1.4f)

- OpenHands runtime cost correction (parallel issue in `openhands-runner.ts`)
- Per-user OpenRouter API key isolation (Phase 5 / Billing)
- `TokenUsage.costSource` column (defer to later phase if needed)
- Caching OpenRouter generation results across re-runs (not useful — each turn has fresh IDs)
- OpenRouter Credits dashboard integration
- Async / deferred cost backfill (sync is correct for quota integration)
- Budget pre-warning before turn (would need cost prediction — separate concern)

## 11. Migration / rollout

No DB migration, no protocol bump. The fix activates automatically when
`ANTHROPIC_BASE_URL` matches `/openrouter\.ai/i` AND `OPENROUTER_API_KEY`
is non-empty. Existing setups without OpenRouter see zero behavioural
change.

Deploy order:
1. Push code → GitHub Action builds new sandbox image
2. Restart sandboxes in projects to pick up new agent-runner code
3. Verify on the next agent turn: `TokenUsage.costUsd > 0`, badge updates
