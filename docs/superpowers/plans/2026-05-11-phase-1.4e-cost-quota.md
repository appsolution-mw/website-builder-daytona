# Phase 1.4e — Cost-Quota + Hard-Limits — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Cap agent spend with two USD limits — a per-user per-day quota enforced at run dispatch (host pre-flight), and a per-turn cap enforced by the broker via the existing `agent.usage` event stream (mid-turn abort via the existing `AbortController` path). Surface live cost state in an always-on workspace topbar badge with green / yellow / red ramp; gate Send at 100 %.

**Architecture:** Two enforcement points, both reading USD caps from env vars (`DEFAULT_DAILY_USD_CAP`, `DEFAULT_PER_TURN_USD_CAP`). The broker side reads `BROKER_PER_TURN_USD_CAP` (forwarded by the worker-pool provisioner) and observes streaming `agent.usage` events inside `executeAgentRun`; on cap exceeded it emits `agent.error` and throws `CostCapExceededError` so the existing ws-server abort path tears the run down. The host side sums `TokenUsage.costUsd` per user per UTC day and rejects new dispatches with HTTP 429 when the daily cap is hit. Browser receives the current quota state in the workspace hydration payload and refetches `/api/projects/[id]/usage` on every `agent.done` WS event to keep the badge live.

**Tech Stack:** Node 24, TypeScript, Prisma 7 + Postgres (Decimal aggregation), Vitest, Next.js 16 App Router, React 19, Tailwind.

**Spec:** [docs/superpowers/specs/2026-05-11-phase-1.4e-cost-quota-design.md](../specs/2026-05-11-phase-1.4e-cost-quota-design.md)

## Plan note vs. spec

Spec §9 sketched a `usage.daily_total` event broadcast from host to browser. The browser↔ws-proxy↔broker topology in this repo does not provide a direct host→browser channel (broker is the only producer of WS events). The plan therefore replaces the new event type with a simpler scheme: the browser refetches `/api/projects/[id]/usage` after every `agent.done` WS event (already a broker-originated event the browser observes). Cost only changes when an agent run completes, so refetch-on-done is equivalent in liveness to a push event and does not introduce a new transport. Spec §5's "no protocol change" guarantee is preserved.

```ts
import type { PrismaClient } from "@prisma/client";

export interface DailyQuotaConfig {
  dailyCapUsd: number;        // 0 = disabled
  perTurnCapUsd: number;      // 0 = disabled
}

export interface DailyQuotaState {
  todaySpend: number;
  dailyCap: number;
  perTurnCap: number;
  resetsAt: string;           // ISO-8601, next UTC midnight
  exceeded: boolean;
}

export function readDailyQuotaConfigFromEnv(): DailyQuotaConfig {
  const dailyRaw = process.env.DEFAULT_DAILY_USD_CAP ?? "0";
  const perTurnRaw = process.env.DEFAULT_PER_TURN_USD_CAP ?? "0";
  const dailyParsed = parseFloat(dailyRaw);
  const perTurnParsed = parseFloat(perTurnRaw);
  return {
    dailyCapUsd: Number.isFinite(dailyParsed) && dailyParsed > 0 ? dailyParsed : 0,
    perTurnCapUsd: Number.isFinite(perTurnParsed) && perTurnParsed > 0 ? perTurnParsed : 0,
  };
}

export function startOfUtcDay(now: Date = new Date()): Date {
  return new Date(Date.UTC(
    now.getUTCFullYear(),
    now.getUTCMonth(),
    now.getUTCDate(),
  ));
}

export function nextUtcMidnight(now: Date = new Date()): Date {
  const d = startOfUtcDay(now);
  d.setUTCDate(d.getUTCDate() + 1);
  return d;
}

export interface DailySpendQuery {
  ownerId: string;
  now?: Date;
}

export async function getUserDailySpend(
  prisma: PrismaClient,
  q: DailySpendQuery,
): Promise<number> {
  const startOfDay = startOfUtcDay(q.now);
  const agg = await prisma.tokenUsage.aggregate({
    _sum: { costUsd: true },
    where: {
      project: { ownerId: q.ownerId },
      createdAt: { gte: startOfDay },
    },
  });
  return agg._sum.costUsd ? Number(agg._sum.costUsd) : 0;
}

export async function getDailyQuotaState(
  prisma: PrismaClient,
  ownerId: string,
  now: Date = new Date(),
): Promise<DailyQuotaState> {
  const config = readDailyQuotaConfigFromEnv();
  const todaySpend = await getUserDailySpend(prisma, { ownerId, now });
  return {
    todaySpend,
    dailyCap: config.dailyCapUsd,
    perTurnCap: config.perTurnCapUsd,
    resetsAt: nextUtcMidnight(now).toISOString(),
    exceeded: config.dailyCapUsd > 0 && todaySpend >= config.dailyCapUsd,
  };
}
```

```tsx
export interface DailyCostBadgeProps {

  resetsAt: string;

  if (props.dailyCap <= 0) return null;
  const ratio = props.todaySpend / props.dailyCap;
  const color =
    ratio >= 1
      ? "text-rose-600 border-rose-600/40 bg-rose-500/10"
      : ratio >= 0.8
        ? "text-amber-600 border-amber-600/40 bg-amber-500/10"
        : "text-muted-foreground border-border";
  const title = `$${props.todaySpend.toFixed(2)} of $${props.dailyCap.toFixed(2)} today; resets ${new Date(props.resetsAt).toLocaleString()}`;
  return (
    <span
      title={title}
      className={`inline-flex items-center gap-1 rounded-md border px-2 py-0.5 text-xs font-medium ${color}`}
    >
      ${props.todaySpend.toFixed(2)} / ${props.dailyCap.toFixed(2)}
    </span>
  );
}
```
