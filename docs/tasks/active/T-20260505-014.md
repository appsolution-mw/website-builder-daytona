Plan the managed Hetzner worker-pool phase: admin-created Hetzner workers,
per-worker project capacity, slot-based project assignment, and public project
subdomains through Cloudflare-backed wildcard TLS.

## Plan

## Notes

The user confirmed H.1c should go directly live against real Hetzner and
Tailscale APIs.

New scope request:
- Admin UI can create/manage Hetzner servers.
- Each server has a configurable maximum number of projects, for example 10.
- New project creation assigns the project to a server with free slots.
- The sandbox is provisioned on the assigned server.
- Project-to-worker assignment is not sticky. After restart/recreate, a project
  may move to another READY worker with free slots.
- Public access uses a Cloudflare-managed domain with wildcard Let's Encrypt
  certificate and per-project subdomains.
