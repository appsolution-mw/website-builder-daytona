  models, matching Postgres migration, and a schema smoke test.

  because this environment has no `DATABASE_URL`; migration SQL was generated
  from Prisma diff output with a placeholder validation URL.
- Task 8 slice in progress: implement `lib/agent-runs/drain.ts` plus durable
  retry/skip state helpers in `lib/agent-runs/queue.ts`.
- Review fixes in progress: prevent duplicate execution on stale drain claims
  and fail/block runs when the execution adapter throws.
- Task 5 slice in progress: add host API routes and focused route tests for
  durable run enqueue/list/action APIs and project event replay.
- API review fix: retry now only requeues the currently blocking terminal run;
  stale failed/cancelled runs return a 400 instead of restarting out of order.
- Task 7/9 slice in progress: add broker run executor boundary, deprecate
  browser-owned `agent.prompt`, and pass OpenHands conversation persistence
  metadata into the SDK bridge.
- Host-owned drain is now wired through DB-claimed runs to
  worker-agent/broker execute streams. Broker/provider events and fs-tracker
  `file.changed` events during active runs are streamed back for host-side
  event persistence; enqueue/retry/skip APIs trigger drains without waiting for
  the long-running task.
- Project chat UI now enqueues messages through `/api/projects/:id/runs`,
  keeps text input available while a run is active, ignores legacy agent events
  from the broker WebSocket, and polls `/api/projects/:id/events` to replay
  persisted run events after reconnect or browser reload.

## Outcome

Implemented the durable project-level agent queue V1:

- Added persisted queue/run/attempt/event storage and project queue state.
- Added host APIs for enqueue/list/retry/skip/cancel and event replay.
- Added a host-owned drain that claims one project run at a time and executes
  through worker-agent and broker streamed run endpoints.
- Wired broker execution to provider runtimes, including OpenHands SDK
  conversation persistence metadata.
- Persisted broker/provider events and active-run file changes for replay.
- Switched the project chat UI to enqueue runs, allow queued messages while a
  task is active, and replay persisted events after reconnect/reload.
