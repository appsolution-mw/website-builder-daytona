# T-20260509-003 - Redirect SDK auto-memory to /workspace/.claude

Status: Done
Date: 2026-05-09
Owner: AI Agent

Related files:
- container/sandbox/entrypoint.sh
- container/sandbox/agent-runner/src/index.ts

## Scope

Follow-up #3 of the Claude Agent SDK integration. Smoke testing produced two
`agent.policy_violation` events because Claude Code's built-in auto-memory
tried to write user preferences (e.g. detected language) to
`/root/.claude/projects/-workspace/memory/MEMORY.md` and `.../user_language.md`.
The `denyOutsideWorkspace` policy hook correctly rejected those writes because
`/root/.claude/` is outside `/workspace/`, so useful auto-memory was lost
even though the agent itself functioned. We want SDK auto-memory to land
under `/workspace/.claude/`, which the `agent-context` bootstrap merge
already populates with CLAUDE.md / skills / agents.

## Plan

- Confirm via `@anthropic-ai/claude-agent-sdk` source that
  `CLAUDE_CONFIG_DIR` is honoured and overrides the
  `~/.claude` default (`process.env.CLAUDE_CONFIG_DIR ?? join(homedir(), ".claude")`).
- Export `CLAUDE_CONFIG_DIR=/workspace/.claude` for the agent-runner
  process in `container/sandbox/entrypoint.sh`, scoped to that spawn only
  (not the broker).
- `mkdir -p /workspace/.claude` before starting agent-runner to make sure
  the directory exists even on first boot before the bootstrap merge runs.
- Add a one-line startup log in `agent-runner/src/index.ts` that records
  the resolved Claude config dir, so we can verify the redirect from the
  sandbox log without poking at the SDK internals.
- Leave the `denyOutsideWorkspace` PreToolUse hook untouched: blocking
  writes outside `/workspace/` is the correct safety boundary; once the
  SDK writes inside `/workspace/.claude/`, the hook lets them through.

## Notes

- `CLAUDE_CONFIG_DIR` is the dedicated env var; no `HOME=/workspace`
  fallback was needed. Setting `HOME` would also have worked but has
  broader side effects (npm cache, git config, etc.) we prefer to avoid.
- Verification will happen on the next sandbox image rebuild + a smoke
  turn that triggers an auto-memory write; expected outcome is that
  `/workspace/.claude/projects/-workspace/memory/` is created instead
  of policy violations being emitted.
- No agent-runner unit tests need updating: the env var is read by the
  SDK itself, not by our TS code.

## Outcome

Done. `CLAUDE_CONFIG_DIR=/workspace/.claude` is now exported only for
the agent-runner spawn in `container/sandbox/entrypoint.sh`, with a
matching `mkdir -p`. Agent-runner logs the resolved config dir at
startup (`claude-sdk auto-memory dir`). `sh -n entrypoint.sh` parses
clean and `pnpm --filter @wbd/agent-runner test` passes. Smoke
verification of the redirect deferred to the next image rebuild.
