## Scope

Find and fix the chat run path that reports `This operation was aborted` after
the agent has already started doing work.

## Plan

- Trace the chat run execution path and identify the abort source.
- Add a focused regression test for the failure mode.
- Implement a small fix that preserves queue behavior.
- Verify focused tests.

## Notes

The UI shows OpenHands work in progress and preview changes, then the run fails
and blocks the queue with `error: This operation was aborted`.

Root cause: `executeProjectRun` used the same worker-agent request timeout as
short commands. Long-running OpenHands streams could exceed the host timeout
even while the sandbox continued doing useful work.

## Outcome

Added a separate `runTimeoutMs` path for streaming agent executions, configured
by `WORKER_AGENT_RUN_TIMEOUT_MS` with a 30 minute default. Short worker-agent
commands still use `WORKER_AGENT_TIMEOUT_MS`.

Verified with:
- `pnpm test:host lib/runtime/worker-pool/__tests__/agent-client.test.ts`
- `pnpm lint`

DB-backed host tests were not run because the current environment points at
database `wbd`, and the test safety guard requires `TEST_DATABASE_URL` to point
at an isolated test database.
