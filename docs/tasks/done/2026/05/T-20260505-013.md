# T-20260505-013 - Repair local dev database stack

Status: Done
Date: 2026-05-05
Owner: AI Agent

Related files:
- prisma/migrations
- prisma/schema.prisma

## Scope

Diagnose and repair the local development stack error that returned HTTP 500
when creating a template project.

## Plan

- Reproduce the failing project creation request.
- Compare Prisma migration history with the actual local database schema.
- Repair local migration metadata without resetting user data.
- Apply missing migrations and verify project creation reaches provisioning.

## Notes

Root cause investigation found local migration drift after worktree integration:
the local dev database had the older Better Auth/GitHub migration name, already
had the project environment table without the matching migration record, and was
missing the durable queue migrations. The running Next dev process also held a
stale Prisma Client and needed a restart after `prisma generate`.

## Outcome

Repaired local migration metadata without resetting data, marked the existing
project environment migration as applied, applied the missing durable queue and
library snapshot migrations, regenerated Prisma Client, restarted the local dev
stack, verified template project creation returns 201, and removed debug
projects created during verification.
