## Scope

Nutzer berichtet: bei Codex und OpenHands werden Bild-Anhänge nicht vom LLM
gelesen. Investigation ergab zwei unabhängige Ursachen:

1. **Codex** (echter Bug): `codex-runner.ts` ignoriert `opts.attachments` und
   übergibt nur einen String-Prompt an `thread.runStreamed`. Der Prompt-Suffix
   `@<path>` ist nur Text. Codex SDK unterstützt jedoch Multimodal via
   `runStreamed(UserInput[])` mit `{ type: "local_image", path }`.
2. **OpenHands** (Modell-Mismatch): Die TS+Python-Bridge übergibt Bilder
   korrekt als `ImageContent`. Der Default `openrouter:qwen/qwen3-coder:free`
   ist aber `input_modalities=["text"]` — text-only — daher verwirft das LLM
   die Bilder. Fix = Image-Capable-Filter im Modell-Dropdown + neuer Default.

  enthält `"image"`. Server-side Query-Param wird ignoriert. Doku:

- Codex-SDK-Beleg: `@openai/codex-sdk@0.122.0` `index.d.ts`:
  `type UserInput = { type: "text"; text: string } | { type: "local_image"; path: string }`,
