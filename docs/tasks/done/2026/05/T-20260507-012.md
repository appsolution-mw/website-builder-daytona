## Scope

After Watchtower restarts the worker-agent its in-memory `brokerTokens`
map is empty, so every host-issued command for an existing sandbox sees
`409 broker-token-missing`. The token IS persisted on the host
(`SandboxToken`), it just never gets fed back. Add a recovery path so
the host re-attaches the token when needed.

## Plan

- New worker-agent endpoint `POST /sandboxes/:id/attach-token` (HMAC,
  body `{ brokerToken }`). Sets `brokerTokens.set(id, token)` and
  returns 204.
- Extend `AgentClient` interface + real + fake impls with
  `attachSandboxToken(sandboxId, token)`.
- New helper `lib/runtime/worker-pool/with-token-recovery.ts` exposing
  `withTokenRecovery(client, sandboxId, fn)`. On `AgentError` 409
  `broker-token-missing` it looks up an unexpired `SandboxToken` row,
  calls `attachSandboxToken`, and retries `fn` once.
- Wire into:
  - `lib/agent-runs/executor-client.ts` — `executeProjectRun` (drain)
    and `cancelProjectRun`.
  - `lib/projects/broker-git.ts` — `getProjectGitStatus` and
    `pushProjectGitChanges`.

Out-of-scope: persisting tokens on disk inside the worker-agent. The
host-driven re-attach is sufficient and avoids new persistence in the
worker-agent process.

## Notes

- Token rows have `expiresAt`; reject expired rows in the resolver.
- One retry only; if the second call still fails, surface the error.
- Streaming call (`executeProjectRun`) throws the `AgentError` synchronously
  before the stream begins, so the same wrapper works.

## Outcome

Worker-agent restart no longer requires sandbox restart to recover.

- New endpoint `POST /sandboxes/:id/attach-token` (HMAC, body
  `{ brokerToken }`, 204 on success, 400 on empty token).
- `AgentClient` extended with `attachSandboxToken(id, brokerToken)` —
  real impl in `agent-client.ts`, no-op in `fake-agent-client.ts`.
- New helper `lib/runtime/worker-pool/with-token-recovery.ts`. On
  `AgentError(409, "broker-token-missing")` it loads the most recent
  unexpired `SandboxToken` row, calls `attachSandboxToken`, retries
  once. Streaming calls (`executeProjectRun`) are safe to retry — the
  agent-client throws the AgentError before the response stream is
  read, so no events leak across attempts.
- Wired into `executor-client.ts` (executeProjectRun, cancelProjectRun)
  and `broker-git.ts` (getProjectGitStatus, pushProjectGitChanges).
- Side hardening in `executor-client.ts`: extracted
  `userMessage = run.userMessage` and pre-computed `resumeSession` so
  the new closure type-narrows cleanly.
