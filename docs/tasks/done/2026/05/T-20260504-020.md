## Scope

Add a browser-native preview capture mode that lets the user drag a region over the preview iframe, capture the selected screen area, and insert the resulting screenshot as a chat image attachment.

## Plan

- Add test-first geometry helpers for drag rectangle normalization and viewport-to-video crop mapping.
- Add preview toolbar capture mode and iframe overlay selection UI.
- Use `navigator.mediaDevices.getDisplayMedia()` to capture the selected browser surface and crop the selected viewport region into a PNG.
- Reuse the existing chat draft attachment pipeline so screenshots appear and send like pasted images.
- Add focused verification and update the monthly changelog.

## Notes

Browser-native capture requires user permission and cannot force the picker to select the iframe. The UI should guide the user to choose the current tab/window and fail softly if capture is unavailable.

## Outcome

Completed. The preview toolbar now has a capture mode. Users can drag a region over the preview iframe, grant browser-native screen capture permission, and the cropped PNG is inserted into the existing chat draft image attachment list.
