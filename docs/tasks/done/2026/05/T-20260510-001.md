## Scope

Komplettes Entfernen von zwei orthogonalen Code-Pfaden auf einer Live-App
(Hetzner + launchnode.de). Hetzner-Worker-Pool und alle anderen Agent-Runtimes
(claude-code, openai-codex, openhands) bleiben funktionsfähig.

User-Entscheidungen:
- Vercel AI: Adapter + komplettes AI SDK (`@ai-sdk/*`, `ai`) entfernen
- Daytona: Runtime-Code + Doku-Historie (Specs/Plans/DAYTONA_SDK_NOTES); DB-Migrationen bleiben
- Hetzner-Worker-Pool: bleibt aktiv
- Branch: direkt auf main
- Projektname `website-builder-daytona` in package.json bleibt
