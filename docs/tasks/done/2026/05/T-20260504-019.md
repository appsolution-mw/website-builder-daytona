## Scope

Pass chat image attachments to OpenHands as native multimodal image content so the selected vision-capable model can inspect the images directly.

## Plan

- Add failing broker/OpenHands runner tests for structured image handoff.
- Extend the broker provider turn options with image attachment references.
- Keep existing workspace file persistence for tool access.
- Pass an attachment manifest into the OpenHands Python bridge.
- Build OpenHands `Message` content with `TextContent` and `ImageContent` for attached images.
- Add a vision-capability guard and clear error for non-vision models.
- Run focused broker tests and update task/changelog.

## Notes

Delegation strategy: keep implementation local. The change touches one runtime contract across broker, runner, tests, and Python bridge, so splitting would add coordination risk without enough benefit.

## Outcome

Completed. OpenHands turns now receive validated chat image attachments through a generated image-url manifest, while the broker still writes the images into the workspace and appends their paths to the prompt for tool access. The Python OpenHands bridge loads the manifest, checks model vision support, and sends a native multimodal `Message` with `TextContent` and `ImageContent`.
