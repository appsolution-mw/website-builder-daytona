## Outcome

Done.

- Schema: `CommitAuthorKind.ROLLBACK` + `Commit.revertedFromSha` —
  Migration `20260510175542_add_rollback_commit_kind`.
- Protocol: 1.16.0; `git.commit` event widened (nullable `turnId`/`runtime`,
  `authorKind: "AGENT" | "ROLLBACK"`, optional `revertedFromSha`).
- Broker: new `revertToCommit` primitive (`git rm -r --cached --ignore-unmatch .`
  before `git checkout <sha> -- .` to drop orphan files; `byteTruncate`-bounded
  body); HTTP route `/internal/projects/:scope/git/revert` broadcasts
  `git.commit` with `authorKind=ROLLBACK` over WS.
- Host: `brokerRevertToCommit` RPC helper, `POST /api/projects/[id]/commits/[sha]/revert`
  route with idle/ownership/HEAD pre-checks (HEAD by `[createdAt desc, id desc]`),
  Commit row written from broker response.
- UI: shared `useRevertCommit` hook (pure reducer + `performRevert`, 12 unit
  tests), `RevertConfirmDialog` (confirm + 5 error variants), Revert button on
  `CommitDetail` (History) and `AgentTurnCommitChip` (chat-turn). Page wiring in
  `app/project/[id]/page.tsx`.
- Verify: `pnpm lint`, `pnpm build`, `pnpm test` (14+47+9+74+132 = 276 across
  protocol/worker-agent/ws-proxy/agent-runner/broker), `pnpm test:host`
  (391/391) all green. Manual browser smoke deferred — requires running
  sandbox stack (`pnpm dev` + `pnpm dev:worker`).
- Heads-up: live DB still needs `pnpm prisma migrate deploy` for
  `20260510120000_drop_vercel_ai_runtime` AND
  `20260510175542_add_rollback_commit_kind` before the next prod deploy.
