## Scope

Write the approved V1 design for a durable project-level agent queue that keeps
agent tasks running after browser disconnects, persists chat/run events, queues
new messages behind the active project task, and supports retrying failed runs.

## Plan

- Capture the approved product decisions.
- Write a focused architecture spec.
- Complete task bookkeeping and changelog entry.

## Notes

- Approved V1 direction: durable project-level FIFO queue in the existing
  worker-pool/Daytona architecture.
- V1 excludes mid-turn steering; messages submitted during active work are
  queued as follow-up runs.
- Failed and cancelled runs block the project queue until a user retries or
  skips the failed run.
