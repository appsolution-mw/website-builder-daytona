## Scope

Phase 1.4a — first slice of Phase 1.4. Auto-commits one git commit per
successful agent turn, persists a Commit row in the host DB, and exposes a
v0/Lovable-style History mode in the workspace. No rollback, no user-edit
commits, no quota — those are explicit follow-ups (1.4a-B, 1.4a-C, 1.4d).

## Notes

- Phase 1.4 in the original master spec covers four sub-systems (commits,
  pause/resume, archive, quota). Decomposed into 1.4a/b/c/d so each ships
  as a self-contained spec → plan → live-validation cycle.
- Two TDD-caught spec bugs were fixed inline in the plan during execution
  (sanitizeCommitTitle ordering, slice(0,10) arithmetic). Two implementer-
  driven robustness improvements (broader getGitStatus catch, UTF-8-safe
  byteTruncate, post-commit-read try/catch, retry-once on Commit-row
  persist) landed as small fix commits.
- Architecture note: the new `lib/runtime/broker-rpc.ts` is host→broker
  direct HTTP (Bearer auth) — distinct from `lib/projects/broker-git.ts`
  (worker-pool path via HMAC + worker-agent). It uses a 30s AbortController
  timeout and wss→https URL conversion.
- `CommitView.runtime` carries the DB-enum form (e.g. `CLAUDE_CODE`); UI
  components map to the protocol-string form via `dbRuntimeToProtocol`
  before passing to `runtimeLabel`.
