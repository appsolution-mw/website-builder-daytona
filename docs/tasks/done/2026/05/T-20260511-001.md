## Scope

USER-authored Commits persistieren. Schließt die "dirty-tree-blockt-Revert"
Lücke aus 1.4a-B und gibt jedem User-Edit eine Audit-Spur.

## Outcome

Done.

- Schema: `Commit.userEmail` nullable column —
  Migration `20260511010947_add_user_email_to_commit`.
- Protocol: 1.17.0; `git.commit.authorKind` union widened to include `"USER"`.
- Broker: `commitUserEditsIfDirty` primitive (peer to `commitAgentTurn`),
  pure-logic `createUserEditDebouncer` (5s debounce, skip-while-running,
  pendingAfter race, flush), in-memory `createUserCommitQueue` with
  long-poll/ack semantics, env-var identity via
  `BROKER_USER_NAME`/`BROKER_USER_EMAIL`, new HTTP routes
  `GET /git/user-commits/pull` (pre-empts POST-only gate) and
  `POST /git/user-commits/ack`, fs-tracker hooks `external`-source events
  into `userDebouncer.notify()`. Pre-revert flush in `/git/revert` and
  pre-run flush in `executeAgentRun` both via `BrokerHandle.flushUserEdits`.
- Worker-pool: `lib/runtime/worker-pool/runtime.ts` looks up
  `Project.owner.{name,email}` per spawn and injects env vars.
- Worker-agent: new `runUserCommitForwarder` long-poll loop
  (1s→30s exponential backoff, 409→ack, HMAC-signed via shared `sign()`),
  started/cancelled alongside `watchBrokerReadiness` in the sandbox lifecycle.
- Host route: `POST /api/internal/sandboxes/[id]/user-commit` with HMAC auth
  (identical pattern to `broker-ready`), idempotent on `(projectId, sha)`,
  writes USER `Commit` row with `userEmail`, `runtime/modelId/agentRunId/sessionId`
  null.
- UI: `commit-serializer` and `CommitView` extended with `userEmail`,
  page.tsx `git.commit` handler carries `userEmail: null` (live broadcast
  doesn't include it; hydration fills it in), `CommitList` prefixes USER rows
  with `✎`, `CommitDetail` shows commit.userEmail (or "user" fallback)
  instead of "launchnode-agent".
- Verify: `pnpm lint`, `pnpm build`, `pnpm test` (protocol 16, worker-agent 49,
  ws-proxy 9, agent-runner 74, broker 158 = 306), `pnpm test:host` (397/397)
  all green. Manual browser smoke deferred — requires running sandbox stack
  (`pnpm dev` + `pnpm dev:worker`).
- Heads-up: live DB needs `pnpm prisma migrate deploy` for
  `20260511010947_add_user_email_to_commit` before the next prod deploy.
