## Scope

Diagnose why sending a chat message with the OpenHands runtime returns `fetch failed` and propose the smallest reliable fix.

## Plan

- Trace the chat send flow from UI to API route and runtime executor.
- Inspect relevant environment/configuration boundaries without exposing secrets.
- Reproduce or identify the failing fetch boundary.
- Document the root cause and recommended fix.

## Notes

- The failed run for project `cmoum97sv000246lzqljl6clp` has `blockedReason = "fetch failed"` and only queue status/error events, so the failure happens before broker/OpenHands events are streamed.
- `.env` has `RUNTIME_MODE=worker-pool-hetzner` but also `WORKER_AGENT_URL=http://127.0.0.1:4500`.
- Nothing is listening on local port 4500, while the project's Hetzner worker health endpoint responds at `http://100.127.25.102:4500/health`.

## Outcome

Root cause identified: the host executor is using the local `WORKER_AGENT_URL` override instead of the assigned Hetzner worker address. Recommended immediate fix is to remove/comment the local `WORKER_AGENT_URL` override and restart `pnpm dev`; recommended code hardening is to make queued run execution ignore `WORKER_AGENT_URL` for managed Hetzner workers and report unreachable worker URLs more clearly.
