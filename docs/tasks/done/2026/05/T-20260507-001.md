## Scope

Investigate and fix the OpenHands chat run failure that reports `'str' object has no attribute 'hex'` after submitting a chat message.

## Plan

- Trace the OpenHands chat run from UI queue to broker and Python bridge.
- Reproduce or isolate the failing value with focused tests.
- Implement the smallest root-cause fix.
- Verify with targeted tests and record the outcome.

## Notes

OpenHands SDK persistence expects `conversation_id` to be a `uuid.UUID` and
uses `conversation_id.hex` for the persisted conversation directory. The bridge
was passing the CLI string directly.

## Outcome

Converted OpenHands bridge conversation IDs from CLI strings to `uuid.UUID`,
added regression coverage, and hardened run enqueue payload validation so
provider session IDs must be UUIDs before reaching the worker path. Rebuilt the
local `wbd/sandbox:dev` image so restarted local sandboxes include the bridge
fix.
