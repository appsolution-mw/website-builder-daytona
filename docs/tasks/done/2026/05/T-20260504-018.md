## Scope

Research how chat image attachments can be passed to the OpenHands runtime so the selected AI model can inspect and analyze images.

## Plan

- Inspect the existing OpenHands runtime integration points.
- Research current OpenHands attachment and multimodal message support.
- Compare required host, protocol, worker, and runtime changes.
- Summarize a safe implementation approach.

## Notes

Delegation strategy: split external OpenHands capability research and local codebase flow analysis across sub-agents, while the main agent coordinates task tracking and final synthesis.

## Outcome

Completed research with two sub-agents. OpenHands supports native image input through `Message` content blocks with `ImageContent(image_urls=[...])`; the current app sends image Base64 through the host protocol, but the broker only writes images to workspace files and passes their paths as prompt text. Native OpenHands vision support should translate image attachments into data URLs at the OpenHands bridge boundary while keeping workspace files for tool access.
