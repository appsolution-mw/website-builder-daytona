## Scope

Per-Turn und Per-User Cost-Caps gemäß Master-Spec §4.2. Adressiert das
Top-2-Risiko "Agent costs balloon" mit Hard-Limits, kontinuierlicher Cost-
Anzeige im UI, und einem Quota-System das Runaway-Turns beendet.

  Hard-Limit-Verhalten (warn / soft-cap / abort mid-turn), UI-Affordance,
  Reset-Window, Per-User vs Per-Project.
- Spec unter `docs/superpowers/specs/2026-05-11-phase-1.4e-cost-quota-design.md`.

## Outcome

Done.

- Helper: `lib/usage/daily-quota.ts` exposes `startOfUtcDay`,
  `nextUtcMidnight`, `readDailyQuotaConfigFromEnv`, `getUserDailySpend`
  (Prisma SUM aggregation across all the user's projects within the UTC
  day), and the composed `getDailyQuotaState`. 13 unit tests.
- Host: pre-flight check in `POST /api/projects/[id]/runs` returns 429
  `daily_quota_exceeded` when `quota.exceeded`. Workspace hydration
  (`/api/projects/[id]`) and `/api/projects/[id]/usage` both include the
  `dailyQuota` field so the badge seeds correctly on first render.
- Broker: `executeAgentRun` accumulates `agent.usage.costUsd` per turn
  and on `runningCostUsd > perTurnCapUsd` emits an `agent.error` with a
  "Cost cap reached" message then throws `CostCapExceededError`. The
  existing ws-server abort path tears the run down. `perTurnCapUsd`
  threads through `StartBrokerOptions` → `executeInternalRun` → the
  executor; broker reads `BROKER_PER_TURN_USD_CAP` env at boot, which
  the worker-pool provisioner forwards from the host's
  `DEFAULT_PER_TURN_USD_CAP`.
- UI: new `<DailyCostBadge />` in the workspace header (green / yellow ≥
  80 % / red ≥ 100 %); Send is disabled with a "Daily $X.XX quota reached"
  tooltip at 100 %. Browser refetches `/usage` after each `agent.done`
  event so the badge stays live without a new transport.
- No schema migration, no protocol bump.
- Verify: `pnpm lint`, `pnpm build`, `pnpm test` (16 + 49 + 9 + 74 + 160 =
  308 across protocol / worker-agent / ws-proxy / agent-runner / broker),
  `pnpm test:host` (412/412) all green. Manual browser smoke deferred —
  requires running sandbox stack with low caps in `.env`.
