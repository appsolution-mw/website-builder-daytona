## Scope

Follow-up #2 of the Claude Agent SDK integration. When a sandbox container is
manually restarted (`docker restart <name>`), Docker re-assigns host-port
mappings (e.g. 4000/tcp 30077 -> 38881). The host has cached the OLD
`brokerPort` for that sandbox (`WorkerSandbox.brokerPort`). New runs sent to
the broker via that stale port fail with `fetch failed` /
`Opening handshake has timed out`. The host needs to re-converge on the
current ports within a few seconds of either worker-agent or sandbox restart.

## Plan

Implement Option B (piggy-back on the existing broker-ready callback):

- Extend `WatchBrokerReadinessArgs` with optional `previewPort` (the broker
  port is already required) and POST both ports to the host on success.
- Update the two call sites in worker-agent to pass the freshly-inspected
  ports from `dockerClient.listSandboxes()` / `createSandbox()`.
- Update the host endpoint to read the optional ports from the body and
  update the cached `WorkerSandbox.brokerPort` / `previewPort` columns when
  they differ from the stored values. Stays idempotent for unchanged
  payloads and for old worker-agent versions that don't send ports.
- Add a small unit test that exercises the "update port if changed" path on
  the host route plus the new payload field on the worker-agent side.

## Notes

- `agentRunnerPort` does not exist as a separate host-mapped port in this
  codebase: the agent-runner shares the sandbox container with the broker
  and is reached via `brokerPort`. We therefore only sync brokerPort and
  previewPort.
- The host route already gates updates with `updateMany({ where: { id, ... } })`
  so a stale notification for a destroyed/restarted project cannot resurrect
  values for a different sandbox.
- HMAC body is the canonical text, so adding fields automatically takes part
  in the signature.

## Outcome

Done. Worker-agent now reports the freshly-observed `brokerPort` (and
`previewPort` when known) on every broker-ready callback. The host
endpoint reads the optional values, compares to the cached
`WorkerSandbox.brokerPort` / `previewPort`, and updates the row when they
differ — so a manual `docker restart <sandbox>` is reconciled within the
broker-readiness probe window (worst case ~180s, typically <1s once the
broker is back up). Old worker-agent versions and empty bodies are
treated as no-op for the port columns.
