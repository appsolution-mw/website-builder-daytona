# T-20260509-004 - Estimate costUsd when SDK reports zero

## Scope

Follow-up #4 of the Claude Agent SDK integration. On Hetzner with OpenRouter as
the upstream (`ANTHROPIC_BASE_URL=https://openrouter.ai/api`,
`ANTHROPIC_API_KEY=<openrouter_key>`), `TokenUsage.costUsd` is always 0 even on
successful turns while `tokensIn`/`tokensOut` are populated. OpenRouter does
not pass `total_cost_usd` through the Anthropic-API-compat layer, so the SDK
result message lands in the host with `total_cost_usd === 0`.

Token counts remain authoritative (from SDK `result.usage`); cost is now a
best-effort estimate derived from token counts and Anthropic public pricing
when the upstream did not surface a cost.

## Plan

- Add `lib/agents/runtimes/claude-code/model-pricing.ts` with a per-million
  token price table for the Claude 4.x family and an `estimateCostUsd()`
  helper. Normalize OpenRouter's `anthropic/` prefix and the SDK `[1m]`
  context-window suffix in lookups.
- Wire the helper into `persistTurnSubtype()` in
  `lib/agent-runs/executor-client.ts`: when the incoming `agent.done` event
  reports `costUsd === 0` and tokens are non-zero, recompute via
  `estimateCostUsd` using the modelId captured from the prior `agent.session`
  event. Persist token counts + cost on the TokenUsage TURN row alongside the
  subtype.
- Add unit tests for the price helper covering missing model id, unknown
  model, Sonnet baseline, cache pricing, and OpenRouter / SDK suffix
  normalization.

## Notes

- Prices need to be kept in sync with Anthropic's public pricing page when
  Anthropic publishes new tiers.
- When the SDK already returned a non-zero cost (direct Anthropic API), we
  keep it as-is. Only zero-cost rows are recomputed.
- The legacy ws-proxy `recordTokenUsage` path persists costUsd straight from
  the broker; that path is not used by the SDK runner.

## Outcome

Done. Host-side estimation chosen over agent-runner computation: simpler to
ship (no sandbox image rebuild), keeps cost-policy out of other runtimes'
event mappers, and the host already owns TokenUsage persistence.

- `lib/agents/runtimes/claude-code/model-pricing.ts` exports
  `estimateCostUsd()` over a Claude 4.x price table (Opus 4.5/4.7,
  Sonnet 4.5/4.6, Haiku 4.5) with input/output and cache read/write
  tiers. `normalizeModelId()` strips OpenRouter's `anthropic/` prefix and
  the SDK's `[1m]` context-window suffix so lookups work for both.
- `persistTurnSubtype()` in `lib/agent-runs/executor-client.ts` only
  recomputes when the SDK reports `costUsd === 0` and there are non-zero
  tokens (input/output/cache). Direct-Anthropic turns (where the SDK
  reports a real cost) keep the SDK value untouched. The model passed to
  the estimator is the actual model observed via `agent.session`, not the
  requested `run.modelId`, so SDK fallbacks (e.g. Sonnet substituting for
  an unavailable Opus tier) bill correctly.
- `pnpm vitest run lib/agents/runtimes/claude-code/__tests__/model-pricing.test.ts`
  passes (8/8). `pnpm lint` clean.
- SDK type definitions confirm `total_cost_usd: number` is non-nullable
  in `SDKResultSuccess` / `SDKResultError`; OpenRouter satisfies the
  schema by emitting `0` rather than omitting the field, which matches
  what we observed in production.

Caveats: prices need manual refresh when Anthropic publishes new tiers;
unknown model ids return `0` (treated as "no estimate available"); the
legacy ws-proxy `recordTokenUsage` path still trusts the broker-reported
cost verbatim.
